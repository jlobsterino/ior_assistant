# Скрипт «ИОР с нефинансовыми последствиями»

> **Notebook:** [`новая_схема_БЗ_v2/ior_nonfinancial_consequences_v2.ipynb`](../новая_схема_БЗ_v2/ior_nonfinancial_consequences_v2.ipynb)
> **Skill ID:** `ior_nonfinancial_consequences_v2`
> **Категория:** детализация по сущностям ИОР (качественные/нефинансовые последствия)

---

## 1. Краткое описание для LLM-маршрутизатора

Формирует список инцидентов операционного риска с **качественными (нефинансовыми) последствиями** за период.
**Один ряд отчёта = одно нефинансовое последствие** (по `nonfin_impact_sid`).
Каждое последствие имеет **вид качественной потери** (19 значений: репутация, регулятор, жалобы, утечка данных и т.д.) и **класс влияния** (5 уровней: Очень высокий, Высокий, Средний, Низкий, Нет).

**Когда LLM выбирает этот скрипт:** запрос про **репутацию, регулятора, жалобы клиентов, СМИ, утечку данных, угрозы непрерывности, ущерб репутации, качественные/нефинансовые последствия**.

## 2. Триггеры — когда применять

### 2.1. Прямые триггеры (по фразе)

| Триггер | Что распознать |
|---------|----------------|
| «нефинансовые последствия по ИОР» / «качественные потери» | основная тема |
| «репутационные риски / ущерб репутации» | `nonfin_impact_kind_name LIKE '%репутаци%'` |
| «жалобы клиентов по ИОР» / «обращения клиентов» | `LIKE '%жалоб%' OR LIKE '%обращени%'` |
| «освещение в СМИ» / «упоминание в прессе» | `LIKE '%СМИ%'` |
| «воздействие со стороны регулятора» / «регуляторные риски» | `LIKE '%регулятор%'` |
| «предписания надзорных органов» | `LIKE '%предписани%'` |
| «утечка данных / информации» | `LIKE '%утечк%'` |
| «угроза непрерывности деятельности» | `LIKE '%непрерывности%'` |
| «угроза жизни и здоровью» / «травматизм» | `LIKE '%жизни и здоровью%'` |
| «отток клиентов после инцидента» | `LIKE '%отток клиентов%'` |
| «срыв сделки» / «неоказание услуги» | `LIKE '%срыв сделки%' OR LIKE '%неоказание%'` |
| «качественные потери уровня "Высокий"/"Очень высокий"» | `nonfin_impact_influence_class_name = 'Высокий'` |

### 2.2. Контекстные триггеры

| Триггер | Действие |
|---------|----------|
| «несоблюдение прав клиентов» | Применить + кросс с `incdnt_type_lvl_1_name = '3. Нарушение прав клиентов и контрагентов'` |
| «инциденты с угрозой репутации» | Применить + `kind_name LIKE '%репутац%'` |
| «насколько критичны последствия за период» | Применить + группировка по `influence_class_name` |
| «нефин потери уровня "Очень высокий"» | + `influence_class_name = 'Очень высокий'` |

### 2.3. Семантические признаки

Ключевые слова: **репутация, регулятор, надзор, жалоба, СМИ, утечка, непрерывность, отток, срыв, угроза, качественный, нефинансовый, реальный ущерб без денег**.

## 6. Алгоритм

1. Фильтр main по периоду `incdnt_entry_dt`.
2. Выгрузка `incident_nonfin_impact` (4 поля).
3. INNER JOIN main x nonfin_impact по `incdnt_id`.
4. Дедуп `(incdnt_sid, nonfin_impact_sid)`.
5. Сортировка `incdnt_entry_dt, incdnt_sid, nonfin_impact_sid`.

## 9. Семантика ключевых полей

### 9.1. `nonfin_impact_kind_name` — 19 видов качественных потерь (полный список)

| Значение | Бизнес-смысл / Когда возникает |
|----------|----------------------------------|
| **Жалобы и обращения клиентов** | Клиент обратился в банк или ЦБ с претензией |
| **Освещение в СМИ** | Случай попал в прессу/онлайн-медиа |
| **Активное обсуждение в блогах и социальных сетях** | Вирусная реакция в соцсетях |
| **Упоминание в негативном свете со стороны представителей** | Публичная критика от чиновников, экспертов |
| **Ущерб репутации** | Общая оценка репутационного ущерба |
| **Воздействие со стороны регулятора** | ЦБ или другие регуляторы провели проверку, направили запрос |
| **Предписания надзорных и правоохранительных органов** | Получено формальное предписание |
| **Ограничения со стороны судебных и (или) административных** | Судебные ограничения, заморозка операций |
| **Утечка, потеря или искажение защищаемой информации** | ИБ-инцидент с данными |
| **Угроза непрерывности деятельности** | Возможный сбой работы банка |
| **Угроза жизни и здоровью сотрудников** | Травмы, угрозы сотрудникам |
| **Приостановка деятельности в результате неблагоприятного события** | Фактический простой |
| **Срыв сделки и (или) неоказание банковской услуги** | Клиент не получил услугу |
| **Снижение качества предоставления услуг / выполнения** | Деградация SLA |
| **Отток клиентов** | Клиенты уходят из банка |
| **Возникновение источников других типов риска** | ИОР породил доп. риски (кредитный, рыночный) |
| **Использование дополнительного времени персонала** | Сотрудники тратят время на разбор инцидента |
| **Другие качественные потери** | Не подпадает под другие категории |
| _NULL_ | Не указано (ошибка ввода) |

### 9.2. `nonfin_impact_influence_class_name` — 5 уровней влияния

| Значение | Интерпретация |
|----------|----------------|
| **Очень высокий** | Критическое влияние, требует немедленного реагирования высшего руководства |
| **Высокий** | Серьёзное влияние, контроль на уровне УВА/УПР |
| **Средний** | Заметное влияние, штатный контроль |
| **Низкий** | Незначительное влияние |
| **Нет** | Нет фактического влияния (формальная регистрация) |

### 9.3. Связка с типом события инцидента

Часто `nonfin_impact_kind_name` коррелирует с `incdnt_type_lvl_1_name`:
- `'3. Нарушение прав клиентов и контрагентов'` → часто `kind_name='Жалобы и обращения клиентов'` или `'Срыв сделки'`
- `'4. Нарушение кадровой политики'` → `'Угроза жизни и здоровью сотрудников'`
- `'2. Нарушение и сбои систем'` → `'Утечка информации'` или `'Угроза непрерывности'`

## 10. Предупреждения о данных

1. **`nonfin_impact_kind_name` заполнено в 99.27%** — почти всегда есть.
2. **`nonfin_impact_influence_class_name` заполнено в 80.92%** — около 19% записей с NULL.
3. **Объём:** 239 тыс. нефин. последствий на 980 тыс. инцидентов. То есть **~24% инцидентов имеют качественные последствия**.
4. **INNER JOIN** автоматически отсекает инциденты без нефин. последствий — это **бизнес-логика отчёта**.
5. **Не путать с флагами риска** в main (`incdnt_behavior_risk_flag` и т.д.) — флаги это категория ИБ/ИС/поведения/модели; нефин. последствия — конкретные проявления (жалоба, СМИ, регулятор).

## 13. Пограничные случаи

| Случай | Поведение |
|--------|-----------|
| Пустой результат | «За период {begin}–{end} ИОР с нефин. последствиями не зарегистрировано.» |
| Запрос только «уровень Очень высокий» | + `influence_class_name='Очень высокий'` + в ответе сделать акцент на критичности |
| «Все ИОР с репутацией» | Применить + `kind_name LIKE '%репутаци%'` (объяснить что это поиск по подстроке) |
| «Регуляторные инциденты» | + `kind_name LIKE '%регулятор%' OR LIKE '%надзор%'` |
| «ИОР с регулятором и СМИ одновременно» | Группировка: `HAVING COUNT(DISTINCT kind_name) >= 2 AND все нужные kind_name присутствуют` |

---

## 14. Известные проблемы

1. **Один инцидент может иметь несколько нефин. последствий разных видов.** При подсчёте «сколько ИОР с регулятором» использовать `COUNT(DISTINCT incdnt_id)`.
2. **`nonfin_impact_kind_name` может содержать NULL** — встретится примерно в 0.7% записей.
3. **Уровень "Нет"** означает что качественное последствие зарегистрировали, но фактического влияния не оказалось — формальная запись. Это не ошибка.
4. **Регистр в `kind_name`** последовательный (все с большой буквы).

## 15. Примеры flow

### Пример 1: «ИОР с критическим репутационным ущербом за 2025»

Фильтр: `kind_name LIKE '%репутаци%'` + `influence_class_name IN ('Очень высокий','Высокий')` + период.
В ответе: список ИОР с самыми критическими случаями.

### Пример 2: «Сколько было жалоб клиентов на ИОР за месяц»

Фильтр: `kind_name='Жалобы и обращения клиентов'`. В ответе — число и сравнение по месяцам.

### Пример 3: «Утечки данных за квартал»

Фильтр: `kind_name='Утечка, потеря или искажение защищаемой информации'`. Очень высокая чувствительность — LLM должен показать минимум информации (без `incdnt_full_descr_txt`).
