# **Скрипт «Невозможность взыскания задолженности по кредитному продукту»**

**Notebook:** [notebooks/credit_no_way_collect_debt_v2.ipynb]

**Skill ID:** credit_no_way_collect_debt_v2

**Категория:** расчёт финансовых последствий по кредитному продукту

**Статус:** полная расчетная логика (фл / юл ммб / юл ксб). I/O вынесен

в блок PARAMS (без интерактивного ввода). Формулы расчёта РВПС/залога/

коэффициентов на 46 типов отклонений сохранены без изменений. Требует

проверки на реальных кредитных витринах перед боевым использованием.

Единый источник логики — scripts/credit_calc_source.py (проходит

py_compile); ноутбук собирается из него scripts/build_credit_notebook.py.

## **1. Краткое описание для LLM-маршрутизатора**

Рассчитывает **финансовые последствия (прямые/косвенные/потенциальные потери)** по конкретному кредитному продукту в случае невозможности

взыскания задолженности. Покрывает 4 основания (п.3 Таблицы 9):

* пропущен срок предъявления требований;  
* истёк срок исковой давности;  
* заёмщик ликвидирован;  
* невключение/несвоевременное включение в РТК.

**Когда LLM выбирает:** запрос про **расчёт потерь по кредиту, РВПС, невозможность взыскания, списание задолженности, залог** с указанием

ID кредита и параметров. Это НЕ ИОР-отчёт — это калькулятор по одному

кредиту.

## **4. Параметризация I/O и расчётные ветки**

| Интерактивный ввод | Параметр |
| :---- | :---- |
| выбор клиента/кредита/ЦПР/отклонения | PARAMS (client_type, id_credit, _) |
| выбор факторов ОР | PARAMS["factor_codes"] |
| сумма залога/вывода средств | PARAMS["zalog_overact_amount"] и т.д. |
| вид события DRP-10027 (утрата/неоформление) | PARAMS["drp_10027_type"] |

**Расчётные функции:**

* physical_face_calculation — ФЛ  
* juridical_face_calculation — ЮЛ сегмент ММБ  
* juridical_face_ksb_calculation — ЮЛ сегмент КСБ

**Корректировки данных по ЮЛ:**

* get_zalog_jur_data — залог запрашивается напрямую из cred_yul_new_corr  
  (obj_plg_sid, OBJ_PLG_COLLATERAL_COST_UPD_RUB_AMT, agr_cred_id) по  
  host_agr_cred_id + gregor_dt. (Таблицы cred_yul нет; cred_yul_new_corr —  
  единственная, где есть и host_agr_cred_id, и колонки залога.)  
* get_loan_juridical_face_data — из cred_yul_new_corr, без фильтра src_sys_id  
  (его в этой таблице нет; он есть только в cred_yul_old, где нет колонок залога).  
* В КСБ-функции порядок получения данных на дату выдачи перенесён до их  
  использования (устранён доступ к переменной до присвоения);  
  isdigit() -> .isdigit().
