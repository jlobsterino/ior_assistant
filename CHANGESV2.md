# CHANGESV2: Сводка изменений по аналитическим гипотезам и выгрузкам

Этот файл содержит лог изменений, внесенных для исправления порядка вывода отчетов, гипотез и графиков, а также восстановления работы GigaChat, устранения падений агента при прямых запросах гипотез, исправления двойного запуска выгрузки, предотвращения потери точности 19-значных идентификастов инцидентов, отображения реального количества строк в карточке выгрузки, исправления блокировки fallback-механизма, редизайна графиков и сопоставления ID для построчного FAISS-поиска.

## 1. Изменения в коде проекта

### 📄 `backend/IOR_pipeline_search.py`
* **Изменение**: Поддержка текстовых строковых ID при сборке и сопоставлении векторов в FAISS (`build_and_cache_small_index`).
* **Детали**: Ранее мы принудительно перевели все ID-колонки в датафрейме Pandas в строковый тип (`str`), чтобы избежать потери точности float64. Из-за этого при передаче `target_ids` в FAISS-индексатор строковые ключи (например, `'7603758380951207937'`) не сопоставлялись с ключами оригинального векторного словаря `id_to_index`, который ожидал целочисленный тип `int`. Это приводило к тому, что в консоли выводилось «Всего ID передано: X, успешно сопоставлено: 0». Добавлен автоматический каст к `int` при поиске по ключам словаря. Теперь сопоставление работает корректно как для строк, так и для чисел.
* **Ссылка на файл**: [IOR_pipeline_search.py](file:///d:/ior_assistant/backend/IOR_pipeline_search.py)

### 📄 `backend/agent/hypothesis.py`
* **Изменение**: Полный редизайн графика динамики (`generate_dynamics_chart`) под стиль темной темы и автоматический выбор категориального графика (`generate_distribution_chart`).
* **Детали**:
  * **Авто-категориальный график**: Добавлена функция `generate_distribution_chart`. Она ищет категориальные срезы (ТБ, тип события) и строит круговую диаграмму (Pie Chart) для $\le 5$ уникальных категорий или горизонтальный график столбцов для 6-12 категорий.
  * **Скрытие нулевых потерь**: Если финансовые потери равны 0 (типично для удаленных инцидентов), шкала потерь и красная линия скрываются, предотвращая плоский красный график посередине.
  * **Темный стиль**: Вся графика перерисована на темно-синий фон (`#0b0f19`), соответствующий теме интерфейса.
  * **Динамический период**: Группировка адаптируется под размер выборки (до 31 дня — по дням, до 180 — по неделям, иначе — по месяцам).
* **Ссылка на файл**: [hypothesis.py](file:///d:/ior_assistant/backend/agent/hypothesis.py)
